A collection of commonly used terms in software testing and test automation.

---

## A

### **API (Application Programming Interface)**
A set of communication rules that allows different software systems to interact.

### **Assertion**
A statement that checks whether the actual result matches the expected result.

### **Automation Test**
Tests executed using automation tools instead of manually performing steps.

---

## B

### **Bug / Defect**
An error, flaw, or failure in a software product that causes incorrect or unexpected results.

### **Boundary Value**
The minimum, maximum, or edge-case input values used for testing.

---

## C

### **CI/CD (Continuous Integration / Continuous Delivery)**
A process of automatically building, testing, and deploying applications.

### **Coverage**
The extent to which the system is tested, including functional, API, and automation coverage.

---

## D

### **Data-Driven Testing**
A testing method where test data is separated from the test scripts.

### **DOM (Document Object Model)**
A structured representation of HTML elements used to identify web locators.

---

## E

### **End-to-End Test**
A test that validates a full user workflow from start to finish.

### **Exploratory Testing**
Simultaneous learning and testing to find unexpected issues.

---

## F

### **Fixture**
Preconfigured data or setup used for preparing test environments.

### **Framework**
A structured collection of tools, utilities, and patterns used to make test automation scalable.

---

## L

### **Locator**
A selector used to identify UI elements (CSS, XPath, ID, etc.).

### **Logging**
Recording events or information during test execution.

---

## M

### **Manual Testing**
Human-driven testing without using automation tools.

---

## P

### **Page Object Model (POM)**
A design pattern where each page is represented as a class.

### **Performance Testing**
Testing the speed, scalability, and stability of a system.

---

## R

### **Regression Testing**
Re-running tests to ensure recent changes haven't broken existing features.

---

## S

### **Smoke Testing**
A quick test to check if the system is stable enough for further testing.

### **Sprint**
A short development cycle used in Agile methodology.

---

## T

### **Test Case**
A documented set of steps to validate a specific functionality.

### **Test Plan**
A high-level document describing testing scope, strategy, and resources.

### **Test Suite**
A collection of related test cases.

---

## U

### **UI Testing**
Testing the front-end user interface interactions.

### **Unit Testing**
Testing individual components or functions in isolation.

---

## W

### **Workflow**
A sequence of user actions or business processes tested end-to-end.
